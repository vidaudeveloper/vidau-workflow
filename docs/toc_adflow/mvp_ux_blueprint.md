# To C MVP 页面蓝图与交互状态

本蓝图对应核心路径：`一句话/URL -> 脚本 -> 分镜 -> 出片 -> 导出`，目标是 60 秒内让新用户看到第一条可预览广告草稿。

## 1) 页面结构（MVP）

1. `Create`（创建页）
   - 输入框：一句话需求
   - 可选输入：产品 URL
   - 主按钮：`创建第一条广告`
   - 次按钮：查看示例
2. `Workspace`（生成工作区）
   - 顶部：任务进度条（脚本/分镜/视频）
   - 中区：三列卡片（脚本卡、分镜卡、视频卡）
   - 右侧：余额与预计消耗
3. `Preview`（预览导出）
   - 视频预览器
   - 字幕开关、比例切换、导出按钮

## 2) 关键状态定义

### Create 页
- **空态**
  - 标题：`你想做什么广告？`
  - 占位文案：`例如：为便携储能电源做一条15秒户外露营广告`
- **加载态**
  - 按钮 loading：`正在生成脚本...`
  - 禁止重复点击
- **失败态**
  - 提示：`创建失败，请重试`
  - 展示原因（网络/余额/配置）
- **余额不足态**
  - 提示：`余额不足，无法开始生成`
  - CTA：`去充值`

### Workspace 页
- **脚本生成中**
  - 脚本卡 skeleton + 进度条（步骤 1/3）
- **分镜生成中**
  - 分镜卡 skeleton + 进度条（步骤 2/3）
- **视频生成中**
  - 视频卡加载 + 预计剩余时间（步骤 3/3）
- **部分失败**
  - 单卡失败，保留已成功部分
  - CTA：`仅重试失败步骤`
- **全部完成**
  - 启用 `预览并导出`

### Preview 页
- **空态**
  - 当视频尚未完成：显示“视频生成中”
- **完成态**
  - 可播放 + 可下载
- **导出失败态**
  - 提示：`导出失败，请重试`
  - CTA：`重新导出`

## 3) 文案基线

- 主按钮统一：`创建第一条广告`
- 低认知成本反馈：少用技术词，多用结果词
- 错误提示模板：`发生了什么 -> 你可以怎么做`

示例：
- `视频生成超时，你可以先导出脚本，或稍后重试视频生成。`

## 4) 信息层级

- 一级：当前能做什么（主动作）
- 二级：当前进度如何（进度与状态）
- 三级：为什么失败（可执行建议）

## 5) 与现有页面关系

- 当前 [`frontend/index.html`](../../frontend/index.html) 与 [`frontend/app.js`](../../frontend/app.js) 保留作为“专业模式”。
- 新手默认进入 To C 向导页（后续可通过 `TOC_MODE_ENABLED` 进行入口切换）。
